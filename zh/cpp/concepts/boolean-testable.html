template<class B>

concept __boolean_testable_impl =                // 仅用于阐释

    std::convertible_to<B, bool>;
 (C++20 起)
template<class B>

concept boolean-testable =                       // 仅用于阐释
    __boolean_testable_impl<B> &&
    requires (B&& b) {
        { !std::forward<B>(b) } -> __boolean_testable_impl;

    };
 (C++20 起)

仅用于阐释的概念 boolean-testable 指定可转换成 bool 的表达式，和逻辑运算符拥有通常行为（包括短路），即使是对于二个不同的 boolean-testable 类型的要求。

正式而言，为满足仅用于阐释的概念 __boolean_testable_impl ，类型必须不定义任何成员 operator&& 与 operator|| ，而且无可生成的非成员 operator&& 及 operator|| 可对实参依赖查找可见。另外给定使 decltype((e)) 为 B 的表达式 e ， boolean-testable 仅若 bool(e) == !bool(!e) 才得到实现。

相等性保持

若表达式对给定的相等输入产生相等输出，则它保持相等性。

  • 表达式的输入由其操作数组成。
  • 表达式的输出由其结果和表达式所修改的所有操作数（若存在）组成。

在标准概念的规范中，操作数定义为仅包含下列内容的最大子表达式：

  • 标识表达式，及
  • 对 std::move 、 std::forward 与 std::declval 的调用。

每个操作数的 cv 限定与值类别，是通过假设每个模板类型形参代表一个 cv 无限定的非数组对象类型确定的。

进一步要求每个要求保持相等性的表达式都稳定：这种表达式带相同输入对象的二次求值必须拥有相等的输出，而无任何对这些输入对象的显式中间修改。

除非另外提醒，每个用于 requires 表达式中的表达式都要求保持相等性且稳定，而表达式的求值必须只修改其非常操作数。必须不修改常操作数。

注解

boolean-testable 类型的例子包括 bool、 std::true_type、 std::bitset<N>::reference 和 int* 。
